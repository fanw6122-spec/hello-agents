# 我的改造版本：智能旅行助手 Pro

## 改造背景
原项目存在以下问题：
1. 景点坐标不准确：LLM从训练数据推断，地图上位置有偏差
2. 餐饮数据是假的：没有真实餐厅搜索，用户无法实际参考
3. 地图显示混乱：所有景点堆在一起，无法区分哪天去哪里

## 改造目标
让这个工具从「展示用demo」变成「基本可用的产品」

## 具体改动

### 改动1：景点坐标真实化
- 问题：原来坐标是LLM编造的
- 方案：景点搜索完成后，调用amap_maps_geo工具查询真实坐标
- 改动文件：backend/app/agents/trip_planner_agent.py

### 改动2：餐饮数据真实化  
- 问题：原来餐饮是LLM凭空生成的
- 方案：新增餐厅搜索步骤，根据每天第一个景点的坐标，
         调用amap_maps_around_search搜索周边真实餐厅
- 改动文件：backend/app/agents/trip_planner_agent.py

### 改动3：地图按天显示
- 问题：所有景点堆在一起，看不出哪天的
- 方案：每天的景点用不同颜色标记，并加上天数标签
- 改动文件：frontend/src/views/Result.vue

## 技术方案
使用的新工具：
- amap_maps_geo：地名转经纬度
- amap_maps_around_search：周边POI搜索

不改动的部分：
- 整体架构不变
- 数据模型基本不变（可能小幅扩展）
- 前端页面结构不变（只改地图显示逻辑）
